# Задание 3 — Технологический радар для «Будущее 2.0»

## Формат радара

Радар разделён на четыре квадранта:

- **Платформа данных и хранение**  
- **Интеграция и API**  
- **Разработка и эксплуатация (DevOps/Observability)**  
- **Аналитика, BI и ИИ**

И четыре кольца:

- **Adopt** — использовать как основной стандарт.  
- **Trial** — пилотировать на ограниченных сценариях.  
- **Assess** — изучать, возможное применение в будущем.  
- **Hold** — избегать новых внедрений, только поддержка.

## 1. Платформа данных и хранение

### Adopt

- **Облачное колонночное хранилище (BigQuery / Snowflake / Redshift)**  
  - **Сценарии**: витрины для портала самообслуживания, доменные хранилища финтеха и клиник.  
  - **Бизнес-эффект**: масштабируемость отчётности, снижение времени расчётов с часов до минут.

- **Облачное объектное хранилище (S3 / GCS / Azure Blob) как Data Lake**  
  - **Сценарии**: сырые данные (raw), подготовленные слои (processed/curated), обучающие выборки для ИИ.  
  - **Бизнес-эффект**: дешёвое масштабируемое хранилище, возможность хранить историю без агрессивного архивирования.

### Trial

- **Data Mesh-подход (доменные витрины и владение данными в доменах)**  
  - **Сценарии**: финтех, клиники, ИИ, партнёры.  
  - **Бизнес-эффект**: независимое развитие доменов, ускорение вывода продуктов.

- **Feature Store для ИИ (Feast / Vertex AI Feature Store / SageMaker Feature Store)**  
  - **Сценарии**: управление фичами для моделей ИИ в медицине и финтехе.  
  - **Бизнес-эффект**: повторное использование фич, согласованность онлайн/офлайн.

### Assess

- **Lakehouse-подход (Delta Lake / Iceberg / Hudi)**  
  - **Сценарии**: единая платформа для стриминга и batch-обработки.  
  - **Бизнес-эффект**: упрощение стека, но требует зрелости команды.

### Hold

- **SQL Server 2008 как центральное DWH**  
  - **Сценарии**: только источник исторических данных в переходный период.  
  - **Бизнес-эффект**: высокий риск и стоимость поддержки, ограниченные возможности масштабирования.

## 2. Интеграция и API

### Adopt

- **API Gateway + REST/gRPC**  
  - **Сценарии**: интеграция доменов (клиника, финтех, ИИ, партнёры), внешние B2B-интерфейсы.  
  - **Бизнес-эффект**: стандартизованный слой интеграции, упрощение подключения новых сервисов.

- **Стриминговая платформа (Kafka / Pub/Sub / Event Hubs)**  
  - **Сценарии**: события клиник, финтех-транзакции, телеметрия оборудования.  
  - **Бизнес-эффект**: near real-time аналитика и снижение нагрузки на источники.

### Trial

- **Event-driven architecture для ключевых доменов**  
  - **Сценарии**: реакция на события в реальном времени (подозрительные операции, критические статусы оборудования).  
  - **Бизнес-эффект**: повышение скорости реакции и качества сервиса.

### Assess

- **GraphQL для агрегированных запросов витрины**  
  - **Сценарии**: сложные выборки с вариативными полями для портала самообслуживания.  
  - **Бизнес-эффект**: повышение гибкости, но требует управления сложностью схем.

### Hold

- **Моношина ESB с жёсткой оркестрацией (Apache Camel как единый центр интеграций)**  
  - **Сценарии**: только поддержка существующих интеграций до миграции.  
  - **Бизнес-эффект**: снижает скорость изменений, повышает связанность систем.

## 3. Разработка и эксплуатация (DevOps/Observability)

### Adopt

- **Infrastructure as Code (Terraform)**  
  - **Сценарии**: управление облачной инфраструктурой для всех доменов.  
  - **Бизнес-эффект**: воспроизводимость, контроль изменений, снижение ручных ошибок.

- **CI/CD для приложений и пайплайнов данных (GitLab CI / GitHub Actions / Jenkins)**  
  - **Сценарии**: деплой сервисов финтеха, ИИ-моделей, ETL/ELT-пайплайнов.  
  - **Бизнес-эффект**: ускорение релизов, снижение риска ошибок при выкладке.

- **Мониторинг и логирование (Prometheus/Grafana, ELK/EFK, облачные решения)**  
  - **Сценарии**: наблюдаемость за сервисами, ИИ-моделями и пайплайнами данных.  
  - **Бизнес-эффект**: быстрое выявление проблем, сокращение простоя.

### Trial

- **Service Mesh (Istio/Linkerd) для критичных доменов**  
  - **Сценарии**: финтех, интеграция с партнёрами.  
  - **Бизнес-эффект**: единая политика безопасности, трейсинг и ретраи.

### Hold

- **Ручное управление инфраструктурой и конфигурацией**  
  - **Сценарии**: только для временных стендов и экспериментов.  
  - **Бизнес-эффект**: высокий риск, ограниченная масштабируемость.

## 4. Аналитика, BI и ИИ

### Adopt

- **Современный BI-инструмент (Power BI / Tableau / Looker)**  
  - **Сценарии**: портал самообслуживания, отчётность по доменам.  
  - **Бизнес-эффект**: быстрая визуализация, самообслуживание для бизнес-пользователей.

- **ML/AI-стек на Python (scikit-learn, TensorFlow, PyTorch) + управляемые сервисы в облаке**  
  - **Сценарии**: модели для медицины и финтеха.  
  - **Бизнес-эффект**: повышение качества диагностики и скоринга, новые сервисы.

### Trial

- **MLOps-платформы (Kubeflow, MLflow, Vertex AI, SageMaker)**  
  - **Сценарии**: управление жизненным циклом моделей, реплицируемые пайплайны обучения.  
  - **Бизнес-эффект**: систематизация работы ИИ, снижение операционных рисков.

### Assess

- **AutoML для быстрых пилотов**  
  - **Сценарии**: быстрая проверка гипотез без большой команды data scientists.  
  - **Бизнес-эффект**: ускоренное прототипирование.

### Hold

- **Тяжёлые настольные BI-клиенты, завязанные на DWH**  
  - **Сценарии**: только поддержка текущих отчётов до миграции.  
  - **Бизнес-эффект**: плохо масштабируются, требуют участия ИТ.
